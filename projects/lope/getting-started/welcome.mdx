Lope gathers all the available **candidate data** that you have ever collected (e.g _<kbd>LinkedIn</kbd>_, _<kbd>Github</kbd>_, _<kbd>CVs</kbd>_, _<kbd>Interview Transcriptions</kbd>_, _<kbd>Custom ATS Columns</kbd>_), and stores them into an aggegated AI database. It enables you a quick way to search candidates using natural language and provides a comprehesive way to compare candidates with each other.

## Internal Sourcing

Lope is an **Internal Sourcing** tool, meaning that leverages your existing candidate database. The thesis behind this is that most recruiting teams have already talked with a bunch of candidates that already have the right criteria to be selected for a future job opportunities, but there is no an efficient way to re-search them and evaluate them on new job openings. 

## Where Lope fits with your ATS

Lope doesn’t replace your ATS. It plugs into it. You import candidates via <Tooltip tip="A two-way integration between Lope and your ATS/CRM ensures that any change made to a candidate profile is automatically synchronized across both systems. For example, if you update a candidate’s name in Bullhorn, the change is instantly reflected in Lope — and the same applies in reverse." headline="ATS Integrations">2-way integrations</Tooltip> from tools like Bullhorn or Workable, and Lope enriches, searches, and structures that data so you can actually reuse your existing database instead of going straight back to LinkedIn or any headhunting tool that leverages new candidates extraction.
